Automated Pagination

To paginate these Datasets, something you may wish a link following tool like a web crawler to do, use the query string arguments 'page' for the page number and 'per_page' for the number of Sites per page. HTTP Link headers of first, prev, next & last are given to indicate URIs to the first, a previous, a next and the last page.

Example:

{{  base_uri }}?page=7&per_page=50

Assuming 500 Sites, this request would result in a response with the following Link header:

Link:   <{{ base_uri }}?per_page=50> rel="first",
        <{{ base_uri }}?per_page=50&page=6> rel="prev",
        <{{ base_uri }}?per_page=50&page=8> rel="next",
        <{{ base_uri }}?per_page=50&page=10> rel="last"

If you want to page through the whole collection, you should start at first and follow the link headers until you reach last or until there is no last link given. You shouldn't try to calculate each page query string argument yourself.
